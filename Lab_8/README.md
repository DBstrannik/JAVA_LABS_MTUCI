# Лабораторная работа №8: Многопоточная обработка данных с использованием Reflection API и Аннотаций

## Описание работы
В рамках данной лабораторной работы реализована гибкая многопоточная система обработки текстовых данных. В проекте применены продвинутые механизмы языка Java: создание пользовательских аннотаций, динамический анализ структуры классов через Reflection API для автоматического поиска методов обработки, а также использование пула потоков (`ExecutorService`) для параллельного выполнения задач.

## Структура проекта
* `DataProcessor.java` — создание пользовательской аннотации `@DataProcessor` уровня метода, доступной во время выполнения (Runtime).
* `TextFilterProcessor.java` — класс, содержащий методы фильтрации строк (удаление пустых строк, комментариев с символом `#`) и их трансформации в верхний регистр. Методы помечены аннотацией `@DataProcessor`.
* `DataManager.java` — центральный компонент системы, отвечающий за:
  * Загрузку данных из файла (`input.txt`).
  * Регистрацию процессоров обработки.
  * Сканирование методов через рефлексию и параллельный запуск задач в пуле потоков (`FixedThreadPool`).
  * Сохранение отфильтрованных данных в выходной файл (`output.txt`).
* `Main.java` — точка входа в программу, координирующая процесс инициализации, чтения, обработки и записи.

## Принцип работы

### 1. Маркировка и динамический поиск (DataProcessor, TextFilterProcessor)
Вместо жесткого прописывания вызовов методов, система использует подход инверсии управления. Класс `TextFilterProcessor` содержит независимые методы обработки, работающие через Stream API. С помощью Reflection API в методе `processData()` класса `DataManager` происходит динамический опрос зарегистрированных объектов: извлекаются все объявленные методы (`getDeclaredMethods()`) и проверяется наличие аннотации `isAnnotationPresent(DataProcessor.class)`.

### 2. Параллельное выполнение задач (DataManager)
Все найденные аннотированные методы оборачиваются в список задач `List<Callable>` и передаются на исполнение пулу потоков `ExecutorService`. Количество потоков в пуле подстраивается под количество зарегистрированных процессоров через `Executors.newFixedThreadPool()`. 

Метод `executor.invokeAll(tasks)` выполняет параллельный запуск цепочки обработчиков, после чего главный поток собирает результаты работы через объекты `Future` и обновляет общее состояние данных, гарантируя изоляцию потоков при обработке списков строк.

## Запуск

Для компиляции и запуска лабораторной работы скопируйте строку целиком и выполните её в терминале:
javac DataProcessor.java TextFilterProcessor.java DataManager.java Main.java && java -cp . Main
