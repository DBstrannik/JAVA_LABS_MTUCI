# Лабораторная работа №7: Многопоточное программирование в Java

## Описание работы
В рамках данной лабораторной работы изучены продвинутые механизмы многопоточного программирования в Java из пакета `java.util.concurrent`. На практике освоено управление пулами потоков с помощью `ExecutorService`, координация асинхронных задач, сбор результатов вычислений через интерфейсы `Callable` и `Future`, а также использование `CompletionService` для построения систем с ограничением по разделяемым ресурсам.

## Структура проекта
* `Task1.java` — Задание 1 (Вариант 2): Многопоточное вычисление суммы элементов массива с использованием `ExecutorService` и разделением данных на равные части.
* `Task2.java` — Задание 2 (Вариант 2): Многопоточный поиск наибольшего элемента в матрице с разделением обработки между потоками из пула.
* `Task3.java` — Задание 3: Симуляция работы трех грузчиков на складе с ограничением по суммарному переносному весу (150 кг) с использованием `ExecutorService` и `CompletionService`.

## Принцип работы

### 1. Вычисление суммы массива (Task1)
Для минимизации времени расчета исходный числовой массив делится на равные блоки, количество которых соответствует числу потоков в пуле. Каждый поток (`Callable`) параллельно сканирует свой участок и возвращает локальную сумму. Главный поток блокируется до завершения всех подзадач, после чего собирает результаты через объекты `Future` и вычисляет финальную сумму.

### 2. Поиск максимума в матрице (Task2)
Матрица (двумерный массив) разбивается на логические сегменты (строки или блоки строк). Потоки в пуле обрабатывают свои участки независимо друг от друга, выполняя локальный поиск экстремума. После завершения всех потоков их локальные максимумы поступают в главный поток, где путем финального сравнения определяется абсолютно наибольший элемент матрицы.

### 3. Симуляция работы склада (Task3)
Реализована модель синхронизации группы потоков (3 грузчика). Программа координирует одновременный забор товаров со склада. Потоки переносят объекты, учитывая их индивидуальный вес. Механизм `CompletionService` позволяет главному процессу обрабатывать результаты разгрузки по мере их фактической готовности. При достижении лимита общей массы в 150 кг триггерится логика транспортировки и освобождения буфера.
